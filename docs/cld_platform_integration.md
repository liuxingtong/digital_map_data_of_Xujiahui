# CLD v4 × 数据平台集成说明
> 供数据源平台 AI 助手读取，用于将 CLD v4 因果回路图植入平台分析流程。
> 最后更新：2026-03 | 版本：v4-draft

> **📌 平台数据对齐**：本文档为 CLD 理论方案。若需与平台现有数据结构（路网边、街景、POI、用地、人口）对齐落地，请参阅 [CLD平台集成优化方案](./CLD平台集成优化方案.md)。

---

## 一、系统概览

**研究对象**：上海徐汇区徐家汇街道适老化空间改造  
**建模工具**：因果回路图（CLD）v4，共 20 节点、39 条边、6 条闭合回路（3正3负）+1条恶性强化+2条注入链  
**平台作用**：为各 CLD 节点提供空间量化数值，供系统动力学（SD）仿真标定使用

---

## 二、节点分类与数据状态

### A类：可直接从平台数据计算（完整）

| 节点 | 全称 | 数据源 | 计算方式 |
|------|------|--------|----------|
| N04 | 路网连通性 | `road/*.graphml` | β指数（边数/节点数），或 γ指数 |
| N05 | 步行可达性半径 | road + landuse | 等时圈面积（步行15min） |
| N06 | 土地混合度 | `landuse_data/*.geojson` | 250m格网 Shannon 熵（`shannon`字段） |
| N07 | 绿化遮阴覆盖率 | `streetview_images/merged.csv` | `GVI`（vegetation+terrain）+ `Shade_Comfort` 均值 |
| N09 | 车流量代理 | `road/*.graphml` | `motor_ratio_pct` × 均值lanes × maxspeed，归一化至[0,1] |
| N11 | 噪声与污染代理 | `road/*.graphml` | 同N09基础变量，叠加`Motor_Pressure`（街景字段） |
| N03 | 街道界面活跃度 | streetview + POI | `Shannon_H` + `Complexity` + 边中点100m内POI密度，等权合成，归一化 |

### B类：需多源合成（公式待确认）

| 节点 | 全称 | 数据源 | 当前方案 | ⚠️ 待确认 |
|------|------|--------|----------|-----------|
| N01 | 社交场所密度（含医疗POI） | `poi_data/*.csv` | 社交餐饮POI密度 + 文化地标POI密度 + 医疗保健POI密度，**加权合成** | **三类权重比例？（建议等权或1:1:0.8）** |
| N08 | 街道舒适度 | streetview | `GVI` + `Shade_Comfort` + `Ped_Friendly` 合成 | **三项权重？（建议0.4:0.3:0.3）** |
| N14 | ◆空间认知复杂度指数 | streetview + landuse + POI | 街景`Complexity` + 用地`shannon`（250m格网）+ POI密度，归一化合成 | **各项权重？（对应数据清单原N12）** |
| N15 | 认知储备激活潜力 | streetview | `ART_Score`（已有公式：0.3×Fascination+0.25×Being_Away+0.2×Coherence+0.25×Ped_Friendly） | 公式完整，**确认ART_Score字段是否已在merged.csv中**（对应数据清单原N13） |
| N17 | ◆环境生理压力指数 | road + streetview | `Motor_Pressure`（街景）+ lanes×maxspeed（路网）归一化合成 | **归一化方式？（建议min-max，研究区范围内）** |
| N_YP | 年轻人白天在场密度（外生） | `population/*.tif` | `population_age15_59.tif` 栅格值作代理 | **是否需要日间/夜间修正系数？** |
| N_GC | 隔代照料强度（外生） | `population/*.tif` | `population_age65above.tif` × `population_age0_14.tif` 空间乘积，归一化 | **乘积还是取min？建议乘积后归一化** |

### C类：仿真内生变量（平台无法直接计算，需 SD 初始赋值）

| 节点 | 全称 | 初始值 | 说明 |
|------|------|--------|------|
| N02 | 老年人聚集强度 ★ | 用N02_proxy替代（见下） | 运行时由各回路驱动动态演变 |
| N10 | 步行意愿 | 0.5 | 文献默认值，无直接数据 |
| N16 | 主动探索意愿 | 0.5 | 文献默认值，无直接数据 |
| N_IG | 代际互动频率 | 0.4 | 低于中性（当前空间不支持代际接触） |
| N18 | ◆人机共生干预指数（设计变量）| 0.0→仿真输入 | 原型装置尚未部署，仿真中作为控制变量拨动 |
| N19 | 行为数据密度 | 0.0 | 装置部署前为零，由R5回路内生产生 |

> **N02_proxy（聚集强度初始标定）**：用 `population_age65above.tif` 100m栅格值，经KDE平滑后归一化，作为仿真 t=0 时刻的N02初始空间分布。

## 三、回路结构速查

### 闭合正反馈（强化回路）

```
R1 社交活力（闭合）
  N01(+)→N02(+)→N03(+)→N01
  数据驱动节点：N01（B类合成）、N03（A类）；N02为仿真内生

R4 认知恢复（闭合，含±非线性）
  N14(±)→N15(+)→N16(+)→N14
  数据驱动节点：N14（B类）、N15（B类）；N16为仿真内生

R5 人机共生（闭合，含时滞τ）
  N18(+)→N16(+)→N02(+)→N19(+)→[τ]→N18
  全为仿真内生节点；N18为控制变量
```

### 闭合负反馈（平衡回路）

```
B2 拥挤自调节
  N02(↑)→N08(-)→N02(↓)
  数据驱动节点：N08（B类合成）

B3 认知过载（v4新增）
  N14(↑)→N17(+)→N16(-)→N14(↓)
  数据驱动节点：N14（B类）、N17（B类）；N16仿真内生

B4 干预疲劳
  N18(-)→N19→N18
  全为仿真内生节点
```

### 恶性强化（无内置平衡，需外生干预）

```
R⁻ 交通诱导
  N09(+)→N11(+)→N10(-)→N09
  数据驱动节点：N09（A类）、N11（A类）；N10仿真内生
  外部输入：N_YP→N09（+），年轻人通勤强化早晚高峰
```

### 注入链（不独立闭合）

```
R6 代际激活注入链
  路径1：N_YP(外生,+)→N_IG(+)→N15(+) → 向R4注入
  路径2：N18(+)→N_IG(+)→N15(+) → R4↔R5交汇点

R2 步行可达（开放链）→注入N02→R1
R3 环境舒适（开放链）→注入N02→R1

B5 文化摩擦（抑制链，不闭合）
  N_YP(+)→N08(-)→N02(-) → 削弱R1
B6 隔代照料锁定（抑制链，不闭合）
  N_GC(+)→N16(-) → 削弱R4
```

## 四、平台执行任务清单

AI 助手执行时，请按以下顺序处理：

1. **计算A类节点**（直接可算，无需确认）
   - 计算单位：路网边中点（用于空间可视化）和研究区250m格网（用于SD标定取均值）
   - 输出格式：`node_id`, `value_normalized [0,1]`, `lon`, `lat`

2. **暂停，等待人工确认B类节点的权重参数**（见⚠️标注）
   - 若无人工输入，可用括号内建议值作为默认值继续

3. **计算B类节点**，输出同A类格式

4. **输出C类节点初始值表**（固定值，直接写入）

5. **生成SD仿真初始化参数文件**，格式：
   ```json
   {
     "t0_node_values": {
       "N01": <float>, "N02": <float_from_proxy>, "N03": <float>,
       "N04": <float>, "N05": <float>, "N06": <float>,
       "N07": <float>, "N08": <float>, "N09": <float>,
       "N10": 0.5,     "N11": <float>, "N14": <float>,
       "N15": <float>, "N16": 0.5,     "N17": <float>,
       "N18": 0.0,     "N19": 0.0,
       "N_GC": <float>, "N_IG": 0.4,  "N_YP": <float>
     }
   }
   ```
