## 一、分析链路总览

```
CLD 框架本地化 → 系统动力学量化 → 图论路网建模 → 多目标优化
```

### Step 1：CLD 框架本地化

- **目标**：将定性因果回路转化为徐家汇场景的量化框架
- **输入**：六条因果回路
- **输出**：带有初始权重的有向因果图
- **数据来源优先级**：① 文献效应量 ② 本地数据回归校准 ③ 专家打分兜底

### Step 2：系统动力学量化

- **目标**：为每条 CLD 连接赋予传递函数，构建可模拟时间演化的系统
- **工具**：Vensim 或 Python（scipy + 数值积分）
- **输出**：不同干预强度下，认知储备激活潜力和街道活力的时间演化曲线

### Step 3：图论路网建模

- **目标**：将 CLD 的状态变量映射到物理路网，识别关键干预节点
- **边权重构成**（示例）：
  - R2 路网连通性 × 0.25
  - R3 舒适度（绿视率 + 遮阴率）× 0.25
  - R4 认知复杂度（地标密度 + 界面多样性）× 0.20
  - R1 社交可达性（POI 密度）× 0.15
  - B1 环境生理压力（噪声 + 污染 + 车流）× -0.15
- **图论指标**：介数中心性、可达性等时圈、整合度
- **工具**：OSMnx、NetworkX、Gephi

### Step 4：多目标优化

- **目标**：找到人机模块的最优部署方案
- **目标函数**：认知储备激活覆盖率、认知风险暴露、社会关系连通度、改造成本
- **算法**：NSGA-II（pymoo）
- **输出**：Pareto 最优方案集

## 二、现有数据与路网挂载关系

| 数据源 | 空间形态 | 挂载方式 | 对应建模变量 |
|--------|----------|----------|--------------|
| 街景 | 点 (lon, lat) | 边中点最近点插值 | R3 舒适度（绿视率、遮阴率） |
| POI | 点 | 边中点半径内密度 | R4 认知复杂度、R1 社交可达性 |
| 房价/小区 | 点 | 边中点半径内密度/面积 | 人口分布代理、居住密度 |
| 人口 | 栅格 | 边中点栅格采样 | 老年人口密度 |
| 用地 | 多边形 | 边中点与多边形相交面积 | 绿地面积、设施密度 |
| 路网 | 边 + 节点 | 图结构本身 | R2 连通性、等时圈、介数中心性 |

### 挂载流程示意

```
街景点 ──haversine最近点──► 边 midpoint ──► 边属性: GVI, SVF, Shade_Comfort...
POI点  ──半径内计数/密度──► 边 midpoint ──► 边属性: poi_count, poi_by_category
小区点 ──半径内密度──────► 边 midpoint ──► 边属性: house_density, unit_price_avg
人口栅格 ──rasterio采样──► 边 midpoint ──► 边属性: population_65plus
用地多边形 ──相交面积────► 边 midpoint ──► 边属性: green_area, park_area
路网自身 ───────────────► 节点/边 ─────► length, degree, betweenness...
```

## 三·补充、路网模块可视化设计（2026-03-08 补充）

### 背景说明

> 关键结论：机动车干道（primary/secondary/tertiary）对应高 N09（车流量）和高 N10/N11（噪声/污染），是 CLD 中 B1（负面环境生理压力）的主要来源。OSM 路网边自带的 highway 标签可直接区分「步行网络」与「车流压力区」，不需要任何额外数据源。

### 哪些边属性适合在地图上按边着色展示

| 指标 | CLD 变量 | 可视化形式 | 优先级 | 说明 |
|------|---------|-----------|--------|------|
| `highway` 道路等级 | B1（motor pressure）、R2/R3 | 边按类型分色 | ⭐⭐⭐ | primary/secondary=红，tertiary=橙，residential=黄，footway/path=绿；最直接反映 CLD 分类 |
| `lanes` 车道数 | N09 车流量 → B1 | 边线宽或颜色深浅 | ⭐⭐ | OSM 覆盖率不完整，约 30–50% 有值；车道多=B1压力大 |
| `maxspeed` 限速 | N10/N11 噪声/污染 → B1 | 边热力色 | ⭐⭐ | 覆盖率同样有限，可结合 highway 类型推断默认值 |
| `length` 路段长度 | R2 步行渗透性 | 边颜色（短=深） | ⭐ | 短密=高渗透，长稀=大街块隔断；视觉直觉不强 |

### 哪些指标不适合地图可视化

| 指标 | 原因 |
|------|------|
| `name`（路名）| 文字标注，在 tooltip 里弹出即可 |
| `oneway`（单向）| 影响路由但老年步行分析视觉意义弱 |
| α/β/γ 指数 | 全图单一数字，放指标卡片即可 |
| 平均路段长度 | 全图单一数字，同上 |
| 交叉口密度 | 单值，放指标面板 |

### 指标面板（数字卡片）建议新增

- **步行专用路比例**（footway/path/steps 占总边数 %）→ R2/R3 基线
- **机动车干道比例**（primary/secondary 占总边数 %）→ B1 基线
- **平均路段长度**（步行渗透性）
- **lanes 数据覆盖率**（数据质量参考）

### 模块结构调整方向

- 取消「加载路网」按钮，改为和其他模块一样：读取本地 `road/xujiahui_walk.graphml` 自动加载
- 侧边栏选择「可视化类型」：道路类型 / 车道数 / 限速 / 路段长度
- 主区域：地图（边着色）+ 指标卡片行 + 类型分布图 + 数据预览

## 四、平台应添加的模块

### Phase A：数据扩充模块

| 模块 | 内容 | 优先级 |
|------|------|--------|
| 噪声/交通压力代理层 | 街景机动车像素占比、路网功能等级作为噪声/压力代理 | ⭐⭐⭐ |
| 老年设施 POI 专项层 | 从 POI 提取认知类、运动类、社交类设施 | ⭐⭐⭐ |
| 候选干预节点管理 | 边角空间候选点（面积、位置、可改造难度） | ⭐⭐⭐ |

### Phase B：路网扩展模块

| 模块 | 内容 | 优先级 |
|------|------|--------|
| 综合边权重计算 | R1–R4、B1 合成「老年人友好性权重」 | ⭐⭐⭐ |
| 节点状态向量 | 节点缓冲区聚合 POI、人口、用地、街景 | ⭐⭐⭐ |
| 用地挂载 | 用地多边形与边/节点相交面积（绿地、公园） | ⭐⭐ |
| 小区挂载 | 边中点半径内居住密度、老年人口代理 | ⭐⭐ |

### Phase C：空间分析模块

| 模块 | 内容 | 优先级 |
|------|------|--------|
| 介数中心性计算 | 识别老年人最常经过的瓶颈节点 | ⭐⭐⭐ |
| 等时圈/可达性分析 | 300m 步行范围覆盖居住点/人口 | ⭐⭐⭐ |
| 服务盲区识别 | 300m 内无认知/社交/运动设施的居住区 | ⭐⭐⭐ |
| Space Syntax 整合度 | 街道认知可读性（可选） | ⭐ |

### Phase D：建模与优化模块

| 模块 | 内容 | 优先级 |
|------|------|--------|
| 系统动力学参数标定 | 本地数据回归估计传递函数效应量 | ⭐⭐⭐ |
| 情景仿真 | 不同干预强度下的时间演化 | ⭐⭐ |
| 多目标优化（NSGA-II） | Pareto 方案集求解 | ⭐⭐ |

### Phase E：结果可视化模块

| 模块 | 内容 | 优先级 |
|------|------|--------|
| 边权重热力地图 | 路网边按友好性权重着色 | ⭐⭐⭐ |
| 节点优先级地图 | 气泡=介数中心性，颜色=激活潜力 | ⭐⭐⭐ |
| 服务盲区地图 | 覆盖空白区域热力图 | ⭐⭐⭐ |
| 等时圈叠加地图 | 300m/500m 步行等时圈 | ⭐⭐ |
| Pareto 方案对比 | 多目标优化结果可视化 | ⭐⭐ |
| 仿真曲线 | 激活潜力时间演化 | ⭐ |
